## WORKSPACE BACKUP PROTOCOL (ALL EMPIRE CHATS)

🔒 **Every chat session that does empire work MUST back up to GitHub AND Google Drive before ending.**

### How it works
- Brand website changes go to `main` branch (as always).
- Workspace scripts, changelogs, and key files go to `workspace-backups` branch on GitHub.
- **Redundancy:** A copy of the workspace backup (tarball/zip) must be uploaded to the `EMPIRE_WORKSPACE_BACKUPS` folder on Google Drive (`17dJFlTq_6dZVRckaNewqmbCapLwDggZM`).
- Path format: `workspace/YYYY-MM-DD-<session-name>/`

### What to back up
1. `SITE_CHANGELOG.md` — always, after every session
2. Any new `.py` scripts created this session
3. A `MANIFEST.json` summarizing what was done

### The backup branch
- **Repo:** `priscadezigns9/priscadezignswebsite`
- **Branch:** `workspace-backups`
- **Folder:** `workspace/YYYY-MM-DD-<chat-name>/`

### Why
Each chat has an isolated workspace. When the chat ends, workspace files are gone. GitHub is the only persistent store. Without this protocol, scripts and context are lost forever.

### Reminder
- Do NOT rely on local `backups/` folders — those are in the workspace and disappear too.
- Always push to GitHub. That is the only real backup.
